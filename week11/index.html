MDN: Validating Forms

Client-side form validation helps ensure data submitted meets the requirements set in the form controls before the data is submit to the server. This is important to ensure a good user experience. Form validation are, required fields and specific data format like phone numbers and emails. The client side gives will not submit until the requirements are met.
Types of validation are Built-in form validation, used in HTML5 form validation features, requirements are set in the HTML. Has better performance but is not as customizable. JavaScript validation is coding you create so is very customizable.
Built in form validations has the fields, required: must be filled before submitted, minlength and maxlength:min and max length of text data, min and max: min and max length of numeric data, type: whether data needs to be a number, email, or other type of data, pattern:defines a pattern the data needs to follow. CSS pseudo classes exist for the form elements :valid and :invalid.
Javascript validation helps when you want control over native error messages or need to work with browsers not supporting built-in form validation.

MDN: Using Fetch

Fetch API provides and interface for fetching resources across the network, it uses the fetch() method. Simple example: fetch('http://example.com/movies.json').then(response => response.json()).then(data => console.log(data)); The gets the JSON file across the network and prints it to the console, does not directly return a JSON object do this is resolved with a Response object and the json() method.
