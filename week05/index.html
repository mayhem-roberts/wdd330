Chapter 10: Debugging

System Error - problem occurs with system or external devices the program interacts with.
Programmer Error - program has incorrect syntax or faulty logic, things like a typo or missing syntax can cause this.
User Error â€“ the user has entered data in wrong so the program cannot proceed.

Exceptions return a value that can help deal with the issue. Stack traces give the sequence of error and lines that it occurred. Warnings happen id there is an error in the code but not a big enough issue to stop the program from running. Not all browsers show the same warnings as others.

Strict mode gives more exceptions and warnings, must be put at the top of the script can also be added to a function so it only effects whatever is inside. Gives more detail to what is wrong.

Debugging in the browser, use break point to run through parts of the code. Helpful because the bug does not usually originate at the error.

The alert method can be placed at different parts in the program to help with debugging, it will show an alert and stop the program until the OK is clicked.
